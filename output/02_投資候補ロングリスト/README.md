## 演習の概要

- **所要時間**: 約45分
- **難易度**: 中上級
- **使用ツール**: Claude Code Web（Sonnet 4.6）＋ GitHub Web
- **業界バリエーション**: 小売（RT・リテイル投資先候補）
- **到達率**: 約70%（Agent◎）
- **データ型**: 型B（表形式→スコアリング）

### 業務の課題（要約）
投資候補のロングリスト作成において、地域・売上規模・従業員数・業種等の複数軸の分類を手作業＋Excelで行っており、担当者依存。過去の投資/見送り判定ロジック（50〜100件規模）が文書化されておらず、再利用できない。

### ありたい姿（要約）
過去の投資判定データから抽出されたスコアリングロジック（重み付け・閾値・合否基準）が再利用可能なモデルとして確立。新規候補を入力すると多軸分類＋スコアリングが自動実行され、ランキングが即座に出力される体制。

## 参加者の進め方

1. **準備（15分）**: 手順書§2〜§3に従いGitHubアカウント準備・リポジトリ作成・過去3件（A/B/C）をアップロード
2. **Step1（10分）**: Claude Code Webに過去3件を読み込ませ、CLAUDE.mdにスコアリングロジック（主要4軸・重み付け・閾値・業態例外ルール）を定義
3. **Step2（10分）**: 新規候補5件（カストル小売）を追加アップロードし、CLAUDE.mdに従って1ファイル統合MD（3セクション）を生成
4. **振り返り（10分）**: 手順書§7のチェックポイントと模範出力（ファイル6・7）で到達度確認

参加者がStep2で生成するのは以下1ファイルです（3セクション構成）:

- セクション2: 構造化データ部（参考）— コードブロック内のCSV風テーブル
- セクション3: 過去10件との比較分析 — 境界ケース（コロンバ生活館・サジタ百貨）の個別審議論点

## 到達率70%の内訳

| 項目 | 機械判定可否 |
|---|---|
| 主要4軸（地域・規模・利益率・業種）のスコアリング | ✅ 高精度で機械化 |
| 業態例外ルール（アパレル・百貨店・小粒高収益） | ✅ CLAUDE.mdで明文化可能 |
| 判定3段階（投資/要検討/見送り） | ✅ 閾値ベースで機械判定可能 |
| 定性面（成長性・経営リスク・新業態持続性） | ⚠️ 残り30%の限界・人の判断で補完 |
| 新立地・新業態（過去判例なし） | ⚠️ スコアリング適用外・個別審議枠 |

> 💡 **実務規模での注意**: 本演習は5件規模ですが、実務ではClaude Codeデスクトップ版（ローカル実行）を使えば50〜100件も同じロジックで処理可能です。本教材の3セクション出力構成は件数が増えても変わりません。
