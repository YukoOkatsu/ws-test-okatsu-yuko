## 本フォルダの内容

本フォルダは、AI活用ワークショップの演習ユースケース#1「案件概要資料の要点抽出と標準化」で使用する教材一式です。参加者向けの手順書と、演習に使うダミーデータ4件、および講師配布用の模範出力例2件が含まれています。

> ⚠️ **【研修用模擬資料】本資料は演習目的で作成された架空データです。登場する企業・個人・数値・地域・組織名は全て架空であり、実在の企業・人物・案件とは一切関係ありません。**

## 演習の概要

- **所要時間**: 約45分
- **難易度**: 中上級
- **使用ツール**: Claude Code Web（Sonnet 4.6）＋ GitHub Web
- **業界バリエーション**: 小売（RT・都市型複合チェーン）
- **到達率**: 約65%（Agent◎）

### 業務の課題（要約）
仲介会社から届く案件概要資料はフォーマットや粒度がまちまちで、重要情報を手作業で拾っており1件2〜3時間要している。抽出品質も担当者の経験差でばらつく。

### ありたい姿（要約）
AIに投入するだけで企業情報・財務概要・拠点構成・人員体制・リスク・取引条件の6カテゴリに自動分類され、標準化出力される。1件2〜3時間 → 30分以内に短縮。

## 参加者の進め方

1. **準備（15分）**: 手順書§2〜§3に従いGitHubアカウント準備・リポジトリ作成・過去3件（A/B/C）をアップロード
2. **Step1（10分）**: Claude Code Webに過去3件を読み込ませ、CLAUDE.mdに6カテゴリの抽出テンプレートを定義
3. **Step2（10分）**: 新規1件（ベテルギウス小売）を追加アップロードし、CLAUDE.mdに従って1ファイル統合MD（3セクション）を生成
4. **振り返り（10分）**: 手順書§7のチェックポイントと模範出力（ファイル6・7）で到達度確認
